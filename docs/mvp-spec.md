# 生活智能体文旅体 MVP 产品规格

## 产品定位

生活智能体文旅体是面向城市文旅场景的可演示 MVP（Minimum Viable Product，最小可行产品）：游客端完成“搜、问、订、游、查订单”的核心链路，运营端完成“看指标、管内容、审材料、配流程”的基础协同，商户端完成“看订单、管库存、切营业状态”的工作台能力。

当前版本以本地可运行为优先级，不承诺已接入真实支付、真实地图、真实票务库存或真实身份认证。易变信息如门票、营业时间、价格以官方接口或景区公告为准。

## 目标用户

| 用户 | 目标 | MVP 价值 |
|---|---|---|
| 游客 | 快速得到可信、可执行的玩法 | 真实 POI 检索、AI 兜底行程、票务预约演示、订单状态 |
| 运营人员 | 了解业务指标并处理内容/审核 | 看板、商户列表、知识库、活动、审核工作流 |
| 审核人员 | 处理商户与内容风险 | 待审列表、通过/驳回、风险提示、备注 |
| 商户 | 管理订单和库存 | 营业状态、库存状态、订单核销演示 |
| 管理员 | 演示全量权限 | 可访问游客、运营、审核、商户全部入口 |

## 核心用户故事

| 模块 | 用户故事 | 输入 | 输出 | 失败状态 |
|---|---|---|---|---|
| 首页 | 游客希望看到今日推荐与快捷入口 | 访问首页 | 推荐、AI、票务、地图入口 | 推荐服务不可用时展示 fallback |
| AI 助手 | 游客输入“西湖一日游，带老人，少排队” | 文本问题、偏好 | 行程建议、推荐卡片、工具调用状态 | 无 API key 时返回 deterministic fallback |
| 推荐 | 游客搜索或筛选杭州 POI | 关键词、城市、分类、标签 | 真实 POI 候选、推荐原因 | 空结果提示换关键词 |
| 行程规划 | 游客生成 1-3 日路线 | 天数、预算、同行人、偏好 | 日程卡、费用估算、可替换点 | 数据不足时返回本地模板 |
| 景点详情 | 游客查看 POI 详情 | POI id 或默认景点 | 图片、地址、开放时间、来源提示 | 缺图/缺评分用安全占位 |
| 票务预约 | 游客选择票种、日期、时段、数量 | 票种、日期、时段、游客数 | 待支付订单 | 库存不足、时段不可用、信息缺失 |
| 支付/订单 | 游客完成演示支付 | 待支付订单、支付方式 | 已支付订单、电子凭证 | 支付超时或主动取消 |
| 我的行程 | 游客查看订单和凭证 | 本地订单状态 | 订单列表、行程时间轴 | 无订单时展示空状态 |
| 套餐 | 游客组合票务/酒店/餐饮 | 套餐选择 | 结算清单 | 后续接口未接入时标注演示 |
| 沉浸体验 | 游客查看 AR/VR 场景 | 场景选择 | 热点与讲解入口 | 仅演示，不加载真实 AR 引擎 |
| 地图 | 游客搜索 POI 和路线 | 模式、关键词、图层 | POI 点位、推荐路线 | 地图接口不可用时展示示意地图 |
| 运营看板 | 运营查看指标 | 日期、景区、状态 | KPI、客流、漏斗、告警 | API 不可用时展示 demo 指标 |
| 内容/商户管理 | 运营筛选和新增商户 | 关键词、表单 | 列表、创建/编辑状态 | 表单校验失败提示 |
| 知识库 | 运营管理 FAQ | 分类、发布状态 | FAQ 列表、命中次数 | 空列表提示导入 |
| 活动专题 | 运营管理活动 | 状态、上下架操作 | PV/UV/CTR、活动状态 | 操作失败提示 |
| 审核中心 | 审核人员处理材料 | 审核结果、备注 | 通过/驳回状态 | 缺少备注时禁止驳回 |
| 工作流 | 运营配置流程节点 | 节点、SLA、工具能力 | 流程视图、发布状态 | 当前为演示配置 |
| 商户工作台 | 商户看订单和库存 | 营业状态、库存同步 | 订单列表、库存状态 | 库存同步失败提示 |

## MVP 范围

| 分组 | 必须完成 | 不做范围 |
|---|---|---|
| 游客端 | 首页、AI 助手、推荐、详情、票务预约、演示支付、我的订单、地图、套餐、沉浸体验 | 真实支付扣款、真实短信、真实地图路径计算 |
| 运营端 | 看板、内容/商户、知识库、活动、审核、工作流 | 多租户、审计日志落库、AB 实验平台 |
| 商户端 | 订单、库存、营业状态、审核状态 | 真实分账、真实 PMS/ERP 对接 |
| 技术底座 | 类型、服务层、API client、Node API、AI adapter、localStorage fallback、权限演示、测试 | 生产级认证、数据库迁移、云部署流水线 |

## 数据对象

| 对象 | 说明 | MVP 来源 |
|---|---|---|
| `Poi` | 文旅点位，包含名称、城市、分类、经纬度、来源审计 | `poi-data/usable-pois.json`，前端 fallback 使用真实子集 |
| `City` | 城市元数据，包含 `id/name/adcode` | `poi-data/china-prefecture-cities.json` |
| `TicketOption` | 票种、价格、库存、时段 | 本地演示服务，后续接官方票务 |
| `Order` | 订单、支付、凭证、游客信息 | API 或 `localStorage` fallback |
| `AIResponse` | AI 回答、推荐卡片、工具调用状态、置信度 | provider adapter 或 deterministic fallback |
| `AdminMetrics` | 看板 KPI、客流、漏斗、告警 | API 或 demo 指标 |
| `Role` | visitor/operator/reviewer/merchant/admin | 演示角色切换 |

## 验收标准

| 编号 | 验收项 |
|---|---|
| A1 | 搜索“杭州”或筛选“景点”能返回真实 POI 子集；后端启动后 `/api/pois?cityId=hangzhou` 返回全量杭州 POI。 |
| A2 | 推荐页、景点详情页、地图页不再只依赖 `mockData`，缺图/缺评分/缺开放时间不崩溃。 |
| A3 | 输入“西湖一日游，带老人，少排队”能返回结构化建议、工具调用状态、推荐卡片；无模型 key 时仍可演示。 |
| A4 | 输入“帮我预约雷峰塔上午票”只给票务候选和确认动作，不假装真实支付成功。 |
| A5 | 票务选择、订单确认、演示支付、电子凭证、我的订单形成闭环，刷新后订单尽量不丢失。 |
| A6 | 运营端列表操作有成功/失败/校验状态，不只是静态按钮。 |
| A7 | 角色切换后路由受控：游客不能进入运营后台，商户只能进入商户工作台，管理员可访问全部。 |
| A8 | `npm run build` 通过；如新增测试，`npm test` 通过。 |

## 阶段路线

| 阶段 | 交付 | 验收 |
|---|---|---|
| 0 审计 | 能力与缺口确认 | 输出文件证据和阶段计划 |
| 1 产品规格 | `docs/mvp-spec.md` | 模块输入/输出/失败状态明确 |
| 2 POI 服务 | 类型、服务层、真实数据接入 | 推荐/详情/地图使用真实 POI |
| 3 API 闭环 | `server/`、API client | health/pois/cities/orders/metrics 可访问 |
| 4 AI 助手 | AI adapter、工具状态 | 无 key fallback 可演示 |
| 5 票务订单 | ticket/order 服务 | 待支付到已支付闭环 |
| 6 运营商户 | 可操作后台 | 表单校验、状态变更 |
| 7 权限 | 角色切换、路由保护 | 访问控制生效 |
| 8 质量 | 测试、lazy loading、chunk 拆分 | build/test 通过 |
| 9 交付 | README、`.env.example`、演示脚本 | 本地可按文档启动 |
